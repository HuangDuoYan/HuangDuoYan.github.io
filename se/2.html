2      解题思路

刚开始编写的时候，我也打听过其他同学的思路。他们大多用Java编写，有的直接使用JSON API，有的转成csv之后用CSV API。事实上，题目的核心是“多行输入的样例”，而每个样例，要么是奖牌榜，要么是每日的赛程，结果都是固定的。那么，我就想一步到位，直接按照题目要求的格式保存。

并且注意到，对于赛程命令，题目没明确日期是否有其它格式，所以我假定日期必须是4位月日。那么，我把txt文件的主名也这么设置，不就方便我打开文件了嘛。

因此，首当其冲是准备数据。然后打开输出文件，解析每条命令，将对应的数据追加到输出文件中即可。

3      数据准备

3.1  爬取赛程数据

在CCTV赛程查询页面上，在表格的上面一栏，随便右键一个日期，打开F12检查。跳转到它的事件代码，我立马发现了API调用。我在loadjquery(m);处打断点，m里面的值就是具体的URL了。比如，

https://zy.api.cntv.cn/olympic/getOlyMatchList?itemcode=&startdate=20240808&venue=&medal=&t=jsonp&cb=OM&serviceId=2024aoyun&olyseason=2024S

可以看到，默认情况下，除了开始日期需要我们设置，其它都不需要改变，比如项目编号或者场馆。

打开一看，是JSON格式的数据。但是，细心一点可以看到，开头有OM(，而结尾有);，它们都不符合JSON语法。所以，我要在批量下载的同时把它们删掉。

3.1.1      批量下载JSON文件

我用Copilot生成了下面的Python代码，批量下载到raw文件夹下：

import requests

import json

from datetime import datetime, timedelta

 

# 基础URL

base_url = "https://zy.api.cntv.cn/olympic/getOlyMatchList?itemcode=&startdate={}&venue=&medal=&t=jsonp&cb=OM&serviceId=2024aoyun&olyseason=2024S"

 

# 开始日期和结束日期

start_date = datetime.strptime("20240724", "%Y%m%d")

end_date = datetime.strptime("20240811", "%Y%m%d")

 

# 当前日期

current_date = start_date

 

while current_date <= end_date:

    # 格式化日期

    date_str = current_date.strftime("%Y%m%d")

   

    # 构建URL

    url = base_url.format(date_str)

   

    # 发送请求

    response = requests.get(url)

   

    # 获取响应内容

    content = response.text

   

    # 提取第4个字符开始，倒数第三个字符结束的内容

    extracted_content = content[3:-2]

   

    try:

        # 将提取的内容转换为JSON对象

        json_content = json.loads(extracted_content)

       

        # 文件名

        file_name = f"raw/output_{date_str}.json"

       

        # 将JSON对象保存到文件

        with open(file_name, 'w', encoding='utf-8') as json_file:

            json.dump(json_content, json_file, ensure_ascii=False, indent=4)

       

        print(f"内容已保存到 {file_name} 文件中")

    except json.JSONDecodeError:

        print(f"日期 {date_str} 的数据无法解析为JSON格式")

   

    # 增加一天

    current_date += timedelta(days=1)

3.1.2      去除多余的嵌套属性

随便点开一个看看，发现data这个键嵌套了所有的内容。没必要！下面的total也不需要。按照题目要求，我们关心的是日程内容，而不是总数！再往下一级，matchList才是我们想要的！

接下来，看到每条记录的字段特别多。有的在我眼中是乱码。经过分析，我们只需要保留这些字段来应付题目输出：

字段

说明

"startdatecn"

比赛开始时刻，包含日期。

"itemcodename"

大项。

"title"

比赛名称。

"venuename"

场馆名称

"homeid"

主场编号。如果是数字，说明是个人赛，否则就是国家大写字母缩写，是团队赛。作为判断依据必须保留。

"homename"

主场名称。

"awayname"

客场名称。

所以再让Copilot生成下面的Python代码，提取matchList，并删除多余字段，保存到filtered文件夹中：

import os

import json

 

# 定义要保留的字段

fields_to_keep = ["startdatecn", "itemcodename", "title", "venuename", "homeid", "homename", "awayname"]

 

# 获取raw文件夹中的所有json文件

raw_folder = 'raw'

output_folder = 'filtered'

os.makedirs(output_folder, exist_ok=True)

 

for filename in os.listdir(raw_folder):

    if filename.endswith('.json'):

        filepath = os.path.join(raw_folder, filename)

       

        # 读取JSON文件内容

        with open(filepath, 'r', encoding='utf-8') as file:

            data = json.load(file)

       

        # 提取matchList并保留指定字段

        match_list = data.get('data', {}).get('matchList', [])

        processed_list = [{field: match.get(field, '') for field in fields_to_keep} for match in match_list]

       

        # 写入新的JSON文件

        output_filepath = os.path.join(output_folder, filename)

        with open(output_filepath, 'w', encoding='utf-8') as output_file:

            json.dump(processed_list, output_file, ensure_ascii=False, indent=4)

 

print("处理完成")

3.1.3      整理字段，与题目一致

题目要求的字段和上一步中的对比，分析如下。

字段名

说明

time

要求是hh:mm。所以我们要把startdatecn的日期和秒删掉，再重命名。

sport

对应itemcodename。

name

对应title。如果homeid是字母还必须追加双方国家名称。

venue

忘记改了。

这里进行名称的转换为了好看一些。

使用Copilot，编写下面的Python代码，将文件保存到tidy文件夹下：

import os

import json

 

# 定义文件夹路径

filtered_folder = 'filtered'

tidy_folder = 'tidy'

 

# 确保tidy文件夹存在

os.makedirs(tidy_folder, exist_ok=True)

 

# 遍历filtered文件夹中的每个文件

for filename in os.listdir(filtered_folder):

    if filename.endswith('.json'):

        filepath = os.path.join(filtered_folder, filename)

       

        # 读取JSON文件内容

        with open(filepath, 'r', encoding='utf-8') as file:

            data = json.load(file)

       

        # 处理每个JSON对象

        for item in data:

            # 修改startdatecn字段

            time = item['startdatecn'].split(' ')[1][:5]

            item['time'] = time

            del item['startdatecn']

           

            # 重命名itemcodename字段

            item['sport'] = item.pop('itemcodename')

           

            # 重命名title字段并追加内容

            if item['homeid'] != "" and item['homeid'][0].isalpha():

                item['name'] = f"{item.pop('title')} {item['homename']}VS{item['awayname']}"

            else:

                item['name'] = item.pop('title')

           

            # 删除homeid、homename和awayname字段

            del item['homeid']

            del item['homename']

            del item['awayname']

       

        # 保存修改后的内容到tidy文件夹

        tidy_filepath = os.path.join(tidy_folder, filename)

        with open(tidy_filepath, 'w', encoding='utf-8') as file:

            json.dump(data, file, ensure_ascii=False, indent=4)

3.1.4      生成txt文件

使用Copilot编写Python代码，将最终的每日赛程存到final文件夹中，主名为月日4位数：

import os

import json

 

# 定义文件夹路径

tidy_folder = 'tidy'

final_folder = 'final'

 

# 确保final文件夹存在

os.makedirs(final_folder, exist_ok=True)

 

# 获取tidy文件夹下的所有json文件

json_files = [f for f in os.listdir(tidy_folder) if f.endswith('.json')]

 

# 遍历每个json文件

for json_file in json_files:

    # 读取json文件内容

    with open(os.path.join(tidy_folder, json_file), 'r', encoding='utf-8') as f:

        data = json.load(f)

   

    # 获取json文件名的最后4个字符

    txt_filename = json_file[-9:-5] + '.txt'

   

    # 打开对应的txt文件

    with open(os.path.join(final_folder, txt_filename), 'w', encoding='utf-8') as f:

        # 遍历每条记录并写入txt文件

        for record in data:

            f.write(f"time:{record['time']}\n")

            f.write(f"sport:{record['sport']}\n")

            f.write(f"name:{record['name']}\n")

            f.write(f"venue:{record['venuename']}\n")

            f.write("-----\n")

 

print("所有文件已处理完毕。")

3.2  整理奖牌榜

CCTV上的奖牌榜，一开始我是直接复制到Excel里面，但是发现不符合题目要求！题目要求我们输出国家缩写，而不是中文名！此时，漆廷煜根据汪璟玢老师班上的材料，先整理出了符合题目要求的奖牌榜csv文件。大家可以问他是怎么得来的。于是我再用Copilot编写下面的Python代码，也把它转换成符合题目要求的rank.txt文件：

import csv

 

# 打开并读取rank.csv文件

with open('rank.csv', 'r', encoding='utf-8') as csvfile:

    reader = csv.DictReader(csvfile)

   

    # 打开rank.txt文件以写入

    with open('rank.txt', 'w', encoding='utf-8') as txtfile:

        for row in reader:

            txtfile.write(f"rank{row['排名']}:{row['国家/地区']}\n")

            txtfile.write(f"gold:{row['金牌']}\n")

            txtfile.write(f"silver:{row['银牌']}\n")

            txtfile.write(f"bronze:{row['铜牌']}\n")

            txtfile.write(f"total:{row['总数']}\n")

            txtfile.write("-----\n")

4      设计模块接口，关键代码展示

实际上，本题的关键代码非常简单：

fout << fin.rdbuf();

从功能点角度看，每个功能都对应着一个fin：

l  输出奖牌榜的函数，它的fin就是奖牌榜文件流。

l  输出每日赛程的函数，它的fin由输入的日期来决定打开哪个文本文件。

同时也必须注意到，可执行文件和数据文件根本不在同一个文件夹下！因此要考虑，在文件名前面加相对路径，或者切换工作目录。我这里采用了第二种方式。

一个功能点用一个全局函数。所以接口原型长这样：

/*

* 以下两个接口函数中的 fout，要求主调函数以二进制方式打开，并由主调函数负责关闭。

* 并且这两个函数要求主调函数设置工作目录为 DATA_DIR。

*/

 

void print_medal_cnt(std::ofstream& fout);

 

/*

* 日期必须是 4 位数的月日

* 如果日期对应的 txt 文件不存在，在输出文件中单独一行 ERR_MSG

*/

void print_daily_schd(std::ofstream& fout, const std::string& day);
